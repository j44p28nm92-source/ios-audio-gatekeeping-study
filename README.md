# ios-audio-gatekeeping-study

#### Background: A problem arises when an increasing number of mechanisms disrupt the free market for niche products (in this case, the iOS audio market) in a variety of ways. This ranges from (more or less) incomplete listings to active market manipulation by self-proclaimed experts and questionable, self-serving communities – but also by individual players who profit from the mechanisms.

#
# STUDIES

New Study: "This paper explores the dark side of niche digital platforms, where support forums evolve into tools of shadow market governance. It investigates a toxic forum bubble whose leaders act as a self-styled regulatory police force. Lacking actual technical or musical expertise, these actors rely on expertise-simulation and coercive group dynamics to enforce arbitrary price and design standards. Through active gatekeeping and the manipulation of anonymous power-users, they establish a parasitic ecosystem designed to redirect market visibility and revenue to favored actors while systematically boycotting non-compliant creators."

# Transparency

Regarding data collection transparency: All results are generated using Apple's official public iTunes query API. This API is subject to strict limitations. Frequently used search terms are entered, and the results are stored and updated in a global database. Lists and feeds are then generated by querying this global database, applying global filters as comprehensively as possible to deliver the most relevant results for mobile music production.

### Note: It's noticeable that certain developers and products are extremely difficult to find here, as Apple apparently uses internal filters that are opaque and virtually impossible for outsiders to access. Therefore, it cannot be guaranteed that the currently very extensive data collection is complete.

# APPS

### AudioApp Researcher RT
(release in preparation)

![image](./AudioApp%20Researcher%20RT.png)

RT version is the realtime search without a global data backend. 
This Mac app has three main functions. It's based on Apple's official iTunes Search API.

1. Enter a search term and click 'Search'. The data set will then be loaded if results are available.
2. The explicitly loaded set can be further searched internally by entering new search terms in realtime (RT) without using the button but just typing in new search terms.
3. The Mouse allowas to select the link and a context menue allowas to directly skip to the Apple AppStore and to call other OS functions.

For example, you can first search for a global term like 'Reverb', click 'Search', and then search for 'Room' or 'Convolution' within the results.
The search is case Insensive.

( Note: The listings here on the site were generated with AudioApp Researcher PRO, an offline extension based on a large database. )
